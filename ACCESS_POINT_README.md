# 📡 Access Point Monitor - دليل التشغيل

## نظرة عامة

هذا السكريبت يحول جهاز اللابتوب الخاص بك إلى نقطة وصول (Access Point) لنظام الحضور الذكي.
يراقب الأجهزة المتصلة بالهوت سبوت ويرسل أحداث الاتصال والانفصال للسيرفر لتسجيل الحضور تلقائياً.

## المتطلبات

1. **Python 3.x** مثبت على الجهاز
2. **مكتبة requests**: `pip install requests`
3. **Windows Mobile Hotspot** مفعل
4. **Backend Server** يعمل على `http://localhost:5000`

## خطوات التشغيل

### 1. تشغيل الهوت سبوت

1. اضغط `Win + I` لفتح الإعدادات
2. اذهب إلى **Network & Internet** → **Mobile hotspot**
3. فعّل **Share my Internet connection**
4. تأكد من اسم الشبكة وكلمة المرور

### 2. تشغيل السيرفر (Backend)

```bash
cd backend
npm run dev
```

تأكد من أن السيرفر يعمل على المنفذ 5000

### 3. تشغيل سكريبت نقطة الوصول

```bash
cd graduation-project
python access_point.py
```

## الإعدادات

يمكنك تعديل الإعدادات في بداية ملف `access_point.py`:

```python
# إعدادات الشبكة
SUBNET = "192.168.137."      # IP Range الافتراضي للهوت سبوت في Windows

# إعدادات السيرفر
BACKEND_URL = "http://localhost:5000/api"
AP_API_KEY = "your_access_point_api_key_change_this"  # نفس القيمة في backend/.env

# معرف نقطة الوصول
AP_IDENTIFIER = "AP_101"     # يجب أن يتطابق مع المسجل في قاعدة البيانات
```

### معرفات نقاط الوصول المتاحة في البيانات التجريبية:

| المعرف    | القاعة   | المبنى         |
| --------- | -------- | -------------- |
| `AP_101`  | قاعة 101 | المبنى الرئيسي |
| `AP_102`  | قاعة 102 | المبنى الرئيسي |
| `AP_103`  | قاعة 103 | المبنى الرئيسي |
| `AP_201`  | قاعة 201 | المبنى الفرعي  |
| `AP_202`  | قاعة 202 | المبنى الفرعي  |
| `AP_LAB1` | معمل 1   | مبنى المعامل   |
| `AP_LAB2` | معمل 2   | مبنى المعامل   |
| `AP_LAB3` | معمل 3   | مبنى المعامل   |

## كيف يعمل النظام؟

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   جوال الطالب   │────▶│  Hotspot اللابتوب │────▶│   السيرفر       │
│  (يتصل بالواي فاي) │     │  (access_point.py) │     │   (Backend)     │
└─────────────────┘     └──────────────────┘     └─────────────────┘
                              │                          │
                              │ 1. يكتشف الجهاز          │
                              │ 2. يستخرج MAC Address    │
                              │ 3. يرسل device-connected │
                              │─────────────────────────▶│
                              │                          │ 4. يبحث عن الطالب بـ MAC
                              │                          │ 5. يسجل الحضور
                              │◀─────────────────────────│
                              │ 6. تأكيد                 │
```

## ربط جهاز الطالب بالنظام

لكي يتم تسجيل حضور الطالب تلقائياً:

1. **في لوحة الأدمن**: اذهب إلى **الطلاب** → اختر الطالب → **تعديل**
2. أدخل **MAC Address** الخاص بجوال الطالب في حقل `deviceInfo.macAddress`
3. احفظ التعديلات

### كيف تعرف MAC Address لجوال الطالب؟

- **Android**: الإعدادات → حول الهاتف → الحالة → عنوان Wi-Fi MAC
- **iPhone**: الإعدادات → عام → حول → عنوان Wi-Fi

## مراقبة الحالة في لوحة التحكم

بعد تشغيل السكريبت:

1. اذهب إلى لوحة التحكم (Admin Dashboard)
2. افتح **القاعات** → **نقاط الوصول**
3. سترى القاعة المحددة في `AP_IDENTIFIER` حالتها **"متصل"**

## استكشاف الأخطاء

### ❌ "لا يمكن الاتصال بالسيرفر"

- تأكد من أن السيرفر يعمل: `cd backend && npm run dev`
- تأكد من أن المنفذ 5000 غير مستخدم

### ❌ "Access Point not registered"

- تأكد من أن `AP_IDENTIFIER` في السكريبت يتطابق مع قاعدة البيانات
- شغل `node src/seeders/seed-large.js` لإضافة البيانات التجريبية

### ❌ "Invalid API key"

- تأكد من أن `AP_API_KEY` في السكريبت = `AP_API_KEY` في `backend/.env`

### ❌ الأجهزة لا تظهر

- تأكد من أن الهوت سبوت مفعل
- تأكد من أن `SUBNET` صحيح (الافتراضي في Windows: `192.168.137.`)
- جرب: `arp -a` في CMD لرؤية الأجهزة المتصلة
